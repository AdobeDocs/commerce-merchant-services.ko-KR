# 규칙 추가

규칙을 만들려면 첫 번째 단계는 규칙 편집기를 사용하여 관련 이벤트를 트리거하는 구매자의 쿼리 텍스트에 있는 조건을 정의하는 것입니다. 그런 다음 규칙 세부 사항을 완료하고 결과를 테스트하고 규칙을 게시합니다.

## 1단계: 규칙 추가

1. 관리에서 **마케팅** > SEO &amp; Search > **라이브 검색**.
1. 설정 **범위** 식별하려면 [저장소 보기](https://docs.magento.com/user-guide/configuration/scope.html) 규칙이 적용되는 위치.
1. 을(를) 클릭합니다. **규칙** 탭.
1. 클릭 **규칙 추가** 규칙 편집기를 실행하려면 를 클릭합니다.

   ![규칙 작업 공간](assets/rules-workspace-add-rule.png)

## 2단계: 조건 설명

조건은 이벤트를 트리거하기 위한 요구 사항입니다. 규칙은 최대 10개의 조건과 25개의 이벤트를 가질 수 있습니다.

![규칙 - 규칙 작성](assets/rules-add-workspace.png)

### 단일 조건

1. 아래 *규칙 작성*&#x200B;에서 을(를) 선택합니다. **조건** 을(를) 충족하고 지침에 따라 명령문을 완료합니다.

   * 검색 쿼리 포함 - 쇼핑객의 쿼리에 있어야 하는 텍스트 문자열을 입력합니다. 일치 설정은 구매자의 쿼리가 카탈로그와 일치하는 정도를 결정합니다. 옵션:<br /> 임의 - 쇼퍼 쿼리 텍스트의 모든 부분이 조건과 일치할 수 있습니다.<br />모두 - 모든 구매자의 쿼리가 조건과 일치해야 합니다.
   * 검색 쿼리: - 쇼핑객의 질의와 정확히 일치하는 텍스트 문자열을 입력합니다. 예: &#39;요가 바지&#39; 규칙 `Search query is` 및 일치 `All` 조건은 하나만 있을 수 있습니다.
   * 검색 쿼리 시작 문자 - 쇼퍼 쿼리 시작 부분에 있어야 하는 문자나 텍스트 문자열을 입력합니다.
   * 검색 쿼리 종료 문자 - 쇼퍼 쿼리 끝에 있어야 하는 문자나 텍스트 문자열을 입력합니다.

   결과는 *규칙 테스트* 창 및 의 번호는 우선 순위별로 지정됩니다. 를 사용할 수 있습니다 *행당 결과 수* 오른쪽 위의 슬라이더 를 사용하여 각 행의 제품 수를 변경할 수 있습니다.

   ![규칙 - 단순](assets/rule-simple-test.png)

1. 다른 쿼리를 테스트하려면 *규칙 테스트* 검색 상자 및 누르기 **반환**.
처음에는 테스트 창에서 조건 검색 상자에서 쿼리를 렌더링합니다. 이제 테스트 쿼리 상자에서 쿼리를 렌더링합니다. 테스트 창은 한 번에 하나의 쿼리만 렌더링합니다.

   ![규칙 - 테스트 업데이트](assets/rule-update-test.png)

1. 결과를 원하는 경우 *조건* 검색 상자. 그런 다음 페이지의 아무 곳이나 클릭하여 테스트 창의 결과를 업데이트합니다.
1. 하나의 조건으로 간단한 규칙을 만들려면 3단계로 이동합니다. [이벤트 추가](#events).

### 여러 조건

1. 여러 조건이 있는 규칙을 만들려면 **조건 추가**.
규칙에는 최대 10개의 조건이 있을 수 있습니다. 두 조건을 연결하는 논리 연산자는 현재 조건을 기반으로 합니다 *일치* 설정 기본적으로 *일치* is `All` 논리 연산자는 `AND`.

   ![규칙 - 검색 쿼리에 포함](assets/rules-search-query-contains-and.png)

1. 두 번째 조건을 선택하고 필요한 쿼리 텍스트를 입력합니다.

   ![규칙 조건](assets/rules-add-condition.png)

1. 규칙의 논리를 변경하려면 **일치** 를 설정하여 구매자의 검색 기준이 쿼리 조건과 얼마나 일치해야 하는지 결정합니다. 설정 **일치** 다음 중 하나를 수행합니다.

   * Any - (기본값) 규칙의 모든 논리 연산자가 `OR` 테스트 창에 결과가 나타납니다.
   * 모두 - 규칙의 모든 논리 연산자가 `AND` 테스트 창에 결과가 나타납니다.

   다음 *일치* 값은 여러 조건을 연결하는 데 사용되는 논리 연산자를 결정합니다. 변경 *일치* 를 설정하면 규칙의 모든 논리 연산자가 변경됩니다. 결합할 수는 없습니다 `AND` 및 `OR` 를 반환합니다.
이 예에서 &quot;요가 바지&quot;를 검색하는 대신 &quot;요가&quot; 또는 &quot;바지&quot;를 검색하는 두 개의 별도 쿼리가 있습니다. 이 규칙은 덜 구체적이며 다른 규칙보다 상점 앞에서 더 자주 트리거됩니다.

   ![규칙 - 일치](assets/rules-match.png)

1. 다른 조건을 추가하려면 **조건 추가** 프로세스를 반복합니다.

## 3단계: 이벤트 추가

이벤트는 조건이 충족될 때 검색 결과를 변경하는 작업입니다. 단일 규칙에는 최대 25개의 이벤트가 있을 수 있습니다.

1. 아래 *이벤트*&#x200B;을(를) 선택하고 을(를) 선택합니다. **이벤트** 관련 조건이 충족될 경우 발생합니다.
예를 들어, `Pin a product`. 그런 다음 고정할 제품의 이름을 입력합니다. 도움이 필요한 경우 테스트 창에서 이름을 찾을 수 있습니다.
그런 다음 *위치* 고정된 제품을 표시할 위치입니다. 제품이 테스트 창에서 새 위치로 이동되고 *고정* 미리 보기 배지.

   ![규칙 - 일치](assets/rule-event-pin-product.png)

1. 여러 이벤트의 경우 조건이 충족될 경우 트리거할 다른 이벤트를 선택합니다.

   * 증폭 - 증폭을 선택합니다. 그런 다음 검색 결과에서 상위 위치로 이동할 제품 이름 또는 SKU를 입력합니다. 테스트 창에는 각 강화된 제품에 *활성화됨* 미리 보기 배지.
   * Bury - 검색 결과에서 SKU 아래로 이동합니다. 각 SKU는 *매장됨* 테스트 창의 미리 보기 배지.
   * 제품 고정 - 제품 이름 또는 SKU를 입력합니다. 그런 다음 제품이 나타날 검색 결과에서 위치 를 선택합니다. 제품이 *고정* 테스트 창의 미리 보기 배지.
   * 제품 숨기기 - 검색 결과에서 SKU를 제외합니다.

## 4단계: 세부 정보 완료

여기에 입력한 정보는 [규칙 세부 사항](rules-workspace.md) 패널.

1. 아래 *세부 사항*&#x200B;를 입력한 후 **이름** 참조하십시오.
1. 간단한 설명 입력 **설명** 규칙에 속해 있어야 합니다.
1. 을(를) 입력합니다. **시작 날짜** 및 **종료 날짜** 규칙이 활성화되거나 달력에서 날짜를 선택합니다.

   날짜 범위를 선택하려면 첫 번째 날짜를 클릭하고 드래그하여 범위를 선택합니다.

   ![규칙 - 완료](assets/rule-add-details.png)

## 5단계: 규칙 테스트

1. 테스트 창에서 규칙 결과를 검사합니다.
1. 규칙에 여러 쿼리가 있는 경우 규칙의 영향을 받을 수 있는 각 쿼리를 테스트합니다.

## 6단계: 저장 및 게시

완료되면 를 클릭합니다. **저장 및 게시**.
규칙 작업 공간의 목록에 규칙이 추가됩니다. 활성 규칙이 즉시 적용되지만 스토어의 캐시된 쿼리 결과를 새로 고치는 데 최대 15분이 걸릴 수 있습니다.

## 필드 설명

### 조건(if)

| 조건 | 설명 |
|--- |--- |
| 검색 쿼리 포함 | 쇼핑객의 쿼리에 포함된 문자 또는 텍스트 문자열입니다. 이 조건을 충족하려면 구매자의 쿼리는 단일 문자만 일치해야 합니다. |
| 검색 쿼리: | 쇼핑객의 질의와 정확히 일치하는 텍스트 문자나 문자열입니다. 이 조건을 사용하면 여러 조건이 있는 복합 쿼리를 작성할 수 없습니다. |
| 검색 쿼리 시작 문자 | 쇼핑객의 쿼리는 이 문자나 텍스트 문자열로 시작됩니다. |
| 검색 쿼리가 다음으로 끝남 | 쇼핑객의 쿼리는 이 문자나 텍스트 문자열로 끝납니다. |

### 논리 연산자

| 연산자 | 설명 |
|--- |--- |
| 또는 | (기본값) 논리 연산자입니다. `OR` 두 조건을 비교하고 하나 이상의 조건이 true인 경우 이벤트를 트리거하는 요구 사항을 충족합니다. |
| 및 | 논리 연산자 `AND` 두 조건을 비교하고 두 조건이 모두 참일 경우 이벤트를 트리거하는 요구 사항을 충족합니다. |

### 일치 연산자

| 연산자 | 설명 |
|--- |--- |
| 임의 | 규칙의 모든 논리 연산자로 변경합니다. `OR` 와 는 일치하는 제품 세트를 반환합니다. |
| 모두 | 규칙의 모든 논리 연산자로 변경합니다. `AND` 와 는 일치하는 제품 세트를 반환합니다. |

### 이벤트

| Event | 설명 |
|--- |--- |
| 증폭 | 검색 결과에서 높은 SKU 또는 SKU 범위를 이동합니다. 테스트 검색 결과에 각 태그가 &quot;강화&quot; 미리 보기 배지로 표시됩니다. |
| 버리 | 검색 결과에서 SKU 또는 SKU 범위를 아래로 이동합니다. 테스트 검색 결과에 각 태그가 &quot;묻혀진&quot; 미리 보기 배지로 표시됩니다. |
| 제품 고정 | 단일 SKU를 검색 결과의 특정 위치에 연결합니다. 테스트 검색 결과에 &quot;고정된&quot; 미리 보기 배지가 표시됩니다. |
| 제품 숨기기 | 검색 결과에서 SKU 또는 SKU 범위를 제외합니다. |

### 세부 사항

| 필드 | 설명 |
|--- |--- |
| 이름 | 규칙의 이름입니다. |
| 시작 날짜 | 예약된 경우 규칙의 시작 날짜입니다. |
| 종료 날짜 | 예약된 경우 규칙의 종료 날짜입니다. |
| 설명 | 규칙에 대한 간단한 설명. |
